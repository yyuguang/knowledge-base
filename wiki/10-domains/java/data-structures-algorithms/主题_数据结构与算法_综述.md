# 主题：数据结构与算法 综述

## 当前结论

数据结构与算法不应只按题目记忆，而应围绕“**数据如何表示**、**操作成本如何分析**、**如何选择或设计求解策略**”建立知识网络。对 Java 开发者而言，还需把抽象结构与 JDK 集合、泛型、对象分配和 API 语义对应起来。

## 总体框架

1. **基础**：复杂度分析、递归、抽象数据类型（ADT）、数组与链表的底层差异。
2. **数据结构**：线性表、栈和队列、哈希表、树、堆、图。
3. **算法**：排序、查找、递归与分治、回溯、动态规划、贪心、字符串算法。
4. **问题建模与复盘**：识别约束，选择结构与范式，证明或验证边界，再沉淀可迁移的模板。

## 核心概念

- [[10-domains/java/data-structures-algorithms/foundations/概念_算法复杂度分析]] — 用时间、空间与输入规模评估方案，是比较实现和判断可行性的共同语言。
- [[10-domains/java/data-structures-algorithms/foundations/概念_数据结构抽象与Java实现]] — 区分 ADT 与具体实现，并把结构映射到 Java 集合框架与自定义实现。

## Java 实践约定

- 先说明结构或算法的抽象操作及复杂度，再给出 Java API 或实现细节，避免把 `ArrayList`、`HashMap` 等类名误当作概念本身。
- 示例默认标注输入规模、边界条件、时间/空间复杂度；涉及递归时补充栈深度风险。
- 对 JDK 内部实现、版本差异或性能结论，须在后续页面绑定可核验来源；本总览暂不将其写成固定事实。

## 当前判断

最合适的学习顺序是：先掌握 [[10-domains/java/data-structures-algorithms/foundations/概念_算法复杂度分析]] 与数组/链表等线性结构，再进入树、堆、哈希和图；算法范式则与相应数据结构交叉练习，而非完全串行学习。
