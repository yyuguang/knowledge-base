# 概念：数据结构抽象与 Java 实现

## 定义

抽象数据类型（Abstract Data Type，ADT）定义数据及其可执行操作的语义，例如“栈支持后进先出地压入与弹出”。数据结构实现则规定这些操作如何在内存中组织和执行，例如用连续数组或链式节点实现栈。

## 为什么重要

把 ADT 与实现分开，才能根据访问、插入、删除、排序、内存和并发等约束选择方案，而不是只因熟悉某个 Java 类就直接使用它。

## 使用场景

- 选择 `ArrayList`、`LinkedList`、`ArrayDeque`、`HashMap`、`TreeMap` 或 `PriorityQueue` 等集合前，先确定需要的抽象操作。
- 设计缓存、任务调度、去重、Top K、层序遍历或图遍历等业务/算法组件。
- 在算法题中决定使用数组、链表、哈希表、堆、树或图的表示方式。

## 核心机制

常见的“抽象操作 → 可选实现”对应关系如下：

| 抽象结构 | 关注的操作 | Java 中常见入口 | 实现选择要点 |
| --- | --- | --- | --- |
| 顺序表 / 列表 | 按下标访问、尾部追加、指定位置插入 | `ArrayList`、`LinkedList` | 连续存储利于随机访问；链式结构避免整体搬移但节点开销更高 |
| 栈 / 队列 / 双端队列 | `push`、`pop`、`offer`、`poll` | `ArrayDeque` | 选择语义明确的接口，避免把集合类名当成业务行为 |
| 映射 / 集合 | 查找、去重、键值关联 | `HashMap`、`HashSet`、`TreeMap` | 区分平均查找效率、顺序需求、比较规则和键的相等性契约 |
| 优先队列 | 获取或移除当前最优元素 | `PriorityQueue` | 适合反复取最小/最大值，不等价于完整排序 |

集合的具体性能和细节还会随 JDK 版本、容量、元素类型及访问模式变化；需要源码级或性能结论时应单独给出版本与证据。

## 例子

“最近待处理任务优先”是优先队列 ADT 的需求，可根据优先级比较器使用 `PriorityQueue`；若需求变成“按提交顺序依次处理”，应选择队列语义。二者都可能存放同一种任务对象，但操作约束不同。

## 常见误区

- 把接口、ADT 与某个默认实现混为一谈，例如认为 `List` 必然具备高效随机访问。
- 因为名称相似而忽略语义差异，例如把优先队列当作已经全序排列的列表。
- 未定义元素相等性、比较器或可变键约束，就依赖哈希/有序集合的查找结果。
- 只按单操作复杂度选结构，忽略整体访问模式、内存占用和代码可维护性。
