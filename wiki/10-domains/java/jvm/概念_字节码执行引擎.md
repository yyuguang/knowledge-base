# 概念_字节码执行引擎

## 定义

字节码执行引擎是 JVM 执行子系统核心，负责将 Class 文件中的字节码指令解释/执行为具体操作。采用**基于栈的指令集架构**（零地址指令），区别于 x86 的寄存器架构。

## 运行时栈帧结构

每个方法调用对应一个栈帧：

- **局部变量表**：Slot 存储变量，long/double 占 2 Slot，编译期确定大小
- **操作数栈**：基于栈的指令集，零地址指令格式
- **动态连接**：符号引用在运行时转为直接引用
- **方法返回地址**：正常 / 异常返回
- **附加信息**：调试信息等

### 栈帧可视化：用 jstack 观察方法调用栈

```java
/**
 * 运行后立即用 jstack <pid> 观察栈帧状态
 *
 * javac StackFrameDemo.java && java StackFrameDemo &
 * jps -l                            # 找到 pid
 * jstack <pid> | grep -A 2 "method"  # 查看栈帧
 */
public class StackFrameDemo {
    public static void main(String[] args) throws Exception {
        new StackFrameDemo().methodA(100);
    }

    void methodA(int n) {
        int result = methodB(n * 2);
        System.out.println("Result: " + result + " (hit enter to exit)");
        try { System.in.read(); } catch (Exception e) {}
    }

    int methodB(int n) {
        return methodC(n + 3) * 10;
    }

    int methodC(int n) {
        // 这个方法挂起 60s，给 jstack 足够的观察窗口
        try { Thread.sleep(60_000); } catch (InterruptedException e) {}
        return n;
    }
}
```

jstack 输出节选（栈帧从底到顶）：

```
"main" #1 prio=5 os_prio=0 tid=0x... nid=0x... waiting ...
   java.lang.Thread.sleep(Native Method)
   StackFrameDemo.methodC(StackFrameDemo.java:23)
   StackFrameDemo.methodB(StackFrameDemo.java:19)
   StackFrameDemo.methodA(StackFrameDemo.java:14)
   StackFrameDemo.main(StackFrameDemo.java:10)
```

每个方法占用一个栈帧。最底部是 `main`，最顶部是 `methodC`（正在 sleep）。每个栈帧包含自己的局部变量表和操作数栈，在 `invokevirtual` 发生时 JVM 将调用者的返回地址压入被调用者的栈帧。

## 方法调用与分派

### 解析（编译期确定）
- `invokestatic`：静态方法 | `invokespecial`：构造器/私有/父类方法
- final 方法（非虚方法）

### 分派（运行期确定）
- **静态分派（重载）**：根据参数静态类型决定，编译期多分派
- **动态分派（重写）**：根据接收者实际类型决定，invokevirtual 运行时单分派
- **虚方法表（vtable）+ 接口方法表（itable）**：优化动态分派性能

### 静态分派（重载）：编译期确定版本

重载版本的选择完全由**参数的静态类型（编译期类型）**决定，与运行时实际类型无关。

```java
/**
 * 静态分派（重载）演示
 * 输出：
 *   hello,guy!
 *   hello,guy!
 * 编译器 view: sr.sayHello(man) → sayHello(Human) 版本被选中
 */
public class StaticDispatch {
    static abstract class Human {}
    static class Man extends Human {}
    static class Woman extends Human {}

    public void sayHello(Human guy)   { System.out.println("hello,guy!"); }
    public void sayHello(Man guy)     { System.out.println("hello,gentleman!"); }
    public void sayHello(Woman guy)   { System.out.println("hello,lady!"); }

    public static void main(String[] args) {
        Human man   = new Man();    // 静态类型 Human，实际类型 Man
        Human woman = new Woman();  // 静态类型 Human，实际类型 Woman
        StaticDispatch sr = new StaticDispatch();

        sr.sayHello(man);    // "hello,guy!" — 静态类型 Human 决定重载版本
        sr.sayHello(woman);  // "hello,guy!" — 同上

        // 强转可绕过静态分派：
        sr.sayHello((Man) man);   // "hello,gentleman!"
        sr.sayHello((Woman) woman); // "hello,lady!"
    }
}
```

反编译验证：`javap -verbose StaticDispatch.class` 会发现 `main` 方法中的两条 `sayHello` 调用均指向同一方法签名 `(LStaticDispatch$Human;)V`。

### 动态分派（重写）：invokevirtual 运行期决议

`invokevirtual` 指令的解析步骤：
1. 获取操作数栈顶对象引用的**实际类型** C
2. 在 C 中查找与方法符号引用匹配的方法
3. 未找到则向父类递归查找
4. 找到则校验权限，通过则执行

```java
/**
 * 动态分派（重写）演示
 * 输出：
 *   Son.showMeTheMoney()
 *   Father.showMeTheMoney()
 *   Daughter.showMeTheMoney()
 */
public class DynamicDispatch {
    static abstract class Parent {
        abstract void showMeTheMoney();
    }
    static class Father extends Parent {
        @Override void showMeTheMoney() { System.out.println("Father.showMeTheMoney()"); }
    }
    static class Son extends Father {
        @Override void showMeTheMoney() { System.out.println("Son.showMeTheMoney()"); }
    }
    static class Daughter extends Father {
        @Override void showMeTheMoney() { System.out.println("Daughter.showMeTheMoney()"); }
    }

    public static void main(String[] args) {
        Parent[] actors = { new Son(), new Father(), new Daughter() };
        for (Parent p : actors) {
            p.showMeTheMoney();  // invokevirtual 根据实际类型查找 vtable
        }
    }
}
```

vtable（虚方法表）在类加载的连接阶段创建：每条 `invokevirtual` 调用实际是查 vtable 的固定偏移执行，避免每次递归查找父类链。`invokeinterface` 同理查 itable。

### invokedynamic：Lambda 与 MethodHandle

JDK 7 引入 `invokedynamic`，核心思想：**把方法调用的目标选择权从 JVM 转移到用户代码（Bootstrap Method）**。JDK 8 Lambda 的底层实现正是 invokedynamic + `LambdaMetafactory`。

```java
import java.lang.invoke.*;
import java.util.function.Function;

/**
 * invokedynamic 两个入口：
 * 1. Lambda 表达式（编译生成 invokedynamic + LambdaMetafactory）
 * 2. MethodHandle 显式使用（静态/构造/虚/字段句柄）
 */
public class InvokeDynamicDemo {
    public static void main(String[] args) throws Throwable {
        // ===== 方式 1：Lambda（等价于 invokedynamic） =====
        Function<String, Integer> strlen = String::length;
        System.out.println("Lambda: " + strlen.apply("hello"));

        // 反编译验证: javap -verbose InvokeDynamicDemo.class
        // 会看到: invokedynamic #... bootstrapMethod = LambdaMetafactory

        // ===== 方式 2：MethodHandle（显式 invokedynamic API） =====
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // 2a. 静态方法句柄：Integer.parseInt(String)
        MethodHandle parseInt = lookup.findStatic(
            Integer.class, "parseInt",
            MethodType.methodType(int.class, String.class));
        int result = (int) parseInt.invokeExact("42");
        System.out.println("parseInt: " + result);

        // 2b. 虚方法句柄：String.concat(String)
        MethodHandle concat = lookup.findVirtual(
            String.class, "concat",
            MethodType.methodType(String.class, String.class));
        String s = (String) concat.invoke("Hello, ", "World!");
        System.out.println("concat: " + s);

        // 2c. 构造方法句柄：new String(byte[])
        MethodHandle newString = lookup.findConstructor(
            String.class, MethodType.methodType(void.class, byte[].class));
        String abc = (String) newString.invokeExact(new byte[]{65, 66, 67});
        System.out.println("new String: " + abc);

        // 2d. getter/setter 句柄
        MethodHandle getProp = lookup.findGetter(Point.class, "x", int.class);
        MethodHandle setProp = lookup.findSetter(Point.class, "y", int.class);
        Point p = new Point(1, 2);
        System.out.println("x=" + getProp.invoke(p));
        setProp.invoke(p, 99);
        System.out.println("y=" + p.y);
    }

    static class Point { int x, y; Point(int x, int y) { this.x = x; this.y = y; } }
}
```

### MethodHandle vs Reflection：性能对比

MethodHandle 和反射都能实现动态方法调用，但机制不同：

| 维度 | MethodHandle | Reflection |
|------|-------------|------------|
| 权限检查 | 创建时检查（lookup 阶段） | 每次调用都检查 |
| 类型安全 | 编译期间验证 MethodType | 运行时 Object[] 传参 |
| 内联优化 | JIT 可内联 | 几乎无法内联 |
| 性能 | 接近直接调用 | 慢 1-2 个数量级 |

```java
import java.lang.invoke.*;
import java.lang.reflect.Method;

/**
 * MethodHandle vs Reflection 性能基准测试
 * 典型输出（JDK 8-17 差异不大）：
 *   Direct: ~3ns/call
 *   MethodHandle: ~5ns/call  (接近直接调用)
 *   Reflection: ~40ns/call   (高 10x)
 */
public class MethodHandleVsReflection {
    static final int WARMUP = 10_000;
    static final int ITER = 100_000_000;

    public static int add(int a, int b) { return a + b; }

    public static void main(String[] args) throws Throwable {
        // 预热 JIT
        for (int i = 0; i < WARMUP; i++) {
            add(i, i);
        }

        // 1. 直接调用基准
        long start = System.nanoTime();
        int sum = 0;
        for (int i = 0; i < ITER; i++) {
            sum += add(i, i);
        }
        long directTime = System.nanoTime() - start;
        System.out.printf("Direct:       %,d ns (sum=%d)%n", directTime, sum);

        // 2. MethodHandle
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        MethodHandle mh = lookup.findStatic(MethodHandleVsReflection.class,
            "add", MethodType.methodType(int.class, int.class, int.class));

        sum = 0;
        start = System.nanoTime();
        for (int i = 0; i < ITER; i++) {
            sum += (int) mh.invokeExact(i, i);
        }
        long mhTime = System.nanoTime() - start;
        System.out.printf("MethodHandle: %,d ns (sum=%d) — %.1fx%n",
            mhTime, sum, (double) mhTime / directTime);

        // 3. Reflection
        Method method = MethodHandleVsReflection.class
            .getDeclaredMethod("add", int.class, int.class);

        sum = 0;
        start = System.nanoTime();
        for (int i = 0; i < ITER; i++) {
            sum += (int) method.invoke(null, i, i);
        }
        long refTime = System.nanoTime() - start;
        System.out.printf("Reflection:   %,d ns (sum=%d) — %.1fx%n",
            refTime, sum, (double) refTime / directTime);
    }
}
```
