# 概念_提示工程

## 定义

提示工程（Prompt Engineering）是通过精心设计输入文本来引导 LLM 行为的技术。本质是利用 LLM 的 **In-Context Learning** 能力——模型无需微调，仅通过 prompt 中的示例和指令即可适应新任务。

## 核心技巧全览

### 1. Zero-shot / One-shot / Few-shot

```python
# Zero-shot：不给示例
"文本:Datawhale的AI Agent课程非常棒！\n情感:"

# One-shot：给一个完整示例
"文本:这家餐厅的服务太慢了。\n情感:负面\n\n文本:Datawhale的AI Agent课程非常棒！\n情感:"

# Few-shot：给多个覆盖不同情况的示例
"文本:这家餐厅的服务太慢了。\n情感:负面\n"
"文本:这部电影的情节很平淡。\n情感:中性\n"
"文本:Datawhale的AI Agent课程非常棒！\n情感:"
```

**选择原则**：Zero-shot 用于简单任务；Few-shot（3-5 个示例）用于需要精确格式或特定风格的复杂任务。

### 2. 角色扮演（Role Prompting）

```
你现在是一位资深的Python编程专家。
请解释一下 Python 中的 GIL（全局解释器锁）是什么，要让一个初学者也能听懂。
```

**关键**：角色设定 = 行为约束，隐式定义了回答的语气、深度和格式。

### 3. 上下文示例（In-context Example）

```
我需要你从产品评论中提取产品名称和用户情感。请严格按照下面的JSON格式输出。

评论:这款"星尘"笔记本电脑的屏幕显示效果惊人，但我不太喜欢它的键盘手感。
输出:{"product_name": "星尘笔记本电脑", "sentiment": "混合"}

评论:我刚买的"声动"耳机音质很棒，续航也超出了我的预期！
输出:
```

**作用**：同时示范了输出格式（JSON）和分类逻辑（混合情感）。

### 4. 思维链（Chain-of-Thought, CoT）

```
一个篮球队在一个赛季的80场比赛中赢了60%。
在接下来的赛季中，他们打了15场比赛，赢了12场。两个赛季的总胜率是多少？
请一步一步地思考并解答。

→
第一步: 第一个赛季赢的比赛数 = 80 × 60% = 48场。
第二步: 总比赛数 = 80 + 15 = 95场。总胜利数 = 48 + 12 = 60场。
第三步: 总胜率 = (60 / 95) × 100% ≈ 63.16%。
```

**原理**：将推理过程外化，引导模型逐步展开而非直接跳到最后答案。

### 5. 指令调优 vs 文本补全

```python
# 对"文本补全"模型的提示方式（需要示例"教学"）
"这是一段将英文翻译成中文的程序。\n英文:Hello\n中文:你好\n英文:How are you?\n中文:"

# 对"指令调优"模型的提示方式（可直接下指令）
"请将下面的英文翻译成中文:\nHow are you?"
```

现代 Chat 模型（GPT-4、Qwen-Chat）都是指令调优模型，可直接下达自然语言指令。

## 采样参数控制

| 参数 | 含义 | 取值范围 | 效果 |
|------|------|----------|------|
| **Temperature** | 控制输出随机性 | 0~2 | 0=确定性，1=平衡，>1=更随机 |
| **Top-p** | 核采样：只考虑累积概率≥p 的 token | 0~1 | 0.9=排除长尾低概率 token |
| **Top-k** | 只从概率最高的 k 个 token 中采样 | 正整数 | k=50 是常用值 |

## 结构化输出技巧

Agent 中普遍采用的约束格式：

```
Thought: [你的思考过程]
Action: function_name(arg_name="arg_value")
```

此格式通过正则可稳定解析：`re.search(r"Action: (.*)", llm_output)`.
