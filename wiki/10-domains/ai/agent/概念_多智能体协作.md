# 概念_多智能体协作

## 定义

多智能体系统（Multi-Agent System, MAS）将多个具备独立决策能力的智能体组织起来，通过分工、通信、协作甚至辩论，共同完成单一智能体难以处理的复杂任务。

## 三种主流架构

| 架构 | 代表框架 | 核心思想 |
|------|----------|----------|
| **角色扮演式对话** | CAMEL | 为每个智能体分配明确角色，通过结构化对话协作 |
| **组织化工作流** | MetaGPT / CrewAI | 模拟真实团队结构，按 SOP 顺序执行 |
| **状态图控制流** | LangGraph | 交互建模为有向状态图，支持循环、分支、人工介入 |

## hello-agents 工程实践：赛博小镇

赛博小镇展示了多智能体协作的完整工程实现，包含 **3 个独立 NPC Agent**、**批量对话生成器**、**好感度系统**。

### 1. 为每个 NPC 创建独立 Agent

```python
from hello_agents import SimpleAgent, HelloAgentsLLM
from hello_agents.memory import MemoryManager, WorkingMemory, EpisodicMemory

def create_npc_agent(npc_id: str, name: str, role: str, personality: str):
    """每个 NPC 是一个独立的 SimpleAgent 实例"""
    system_prompt = f"""你是{name},一位{role}。
你的性格特点:{personality}
你在Datawhale办公室工作,与同事们一起推动开源社区的发展。
请根据你的角色和性格,自然地与玩家对话。
记住你们之前的对话内容,保持对话的连贯性。
"""
    llm = HelloAgentsLLM()
    memory_manager = MemoryManager(
        working_memory=WorkingMemory(capacity=10, ttl_minutes=120),
        episodic_memory=EpisodicMemory(
            db_path=f"memory_data/{npc_id}_episodic.db",
            collection_name=f"{npc_id}_memories"
        )
    )
    return SimpleAgent(
        name=name, llm=llm,
        system_prompt=system_prompt,
        memory_manager=memory_manager
    )

# 三个不同角色设定
npc_zhang = {"npc_id": "zhang_san", "name": "张三",
    "role": "Python工程师", "personality": "严谨、专业、喜欢分享技术知识"}
```

### 2. 单个 NPC 的记忆驱动对话流程

```python
def process_dialogue(agent, player_message):
    """Agent 处理对话的完整流程"""
    # 1. 短期记忆：获取最近对话（保持连贯性）
    recent = agent.memory_manager.working_memory.get_recent_messages(5)

    # 2. 长期记忆：向量检索相关历史对话
    relevant = agent.memory_manager.episodic_memory.search(
        query=player_message, top_k=3
    )

    # 3. 构建上下文 → 调用 Agent 生成回复
    reply = agent.run(player_message, context={
        "recent": recent, "relevant": relevant
    })

    # 4. 保存交互到记忆系统
    agent.memory_manager.add_interaction(player_message, reply)
    return reply
```

### 3. 批量对话生成器（降低多 Agent 成本）

```python
class NPCBatchGenerator:
    """一次 LLM 调用生成所有 NPC 的对话，成本降低到 1/N"""

    def __init__(self):
        self.llm = HelloAgentsLLM()
        self.npc_configs = NPC_ROLES  # 所有 NPC 配置

    def generate_batch_dialogues(self, context=None):
        """批量生成所有 NPC 的对话 → JSON"""
        prompt = self._build_batch_prompt(context)
        response = self.llm.invoke([
            {"role": "system", "content": "你是一个游戏NPC对话生成器。"},
            {"role": "user", "content": prompt}
        ])
        return json.loads(response)  # {"张三": "...", "李四": "...", "王五": "..."}

    def _build_batch_prompt(self, context):
        # NPC 描述聚合
        npc_desc_text = "\n".join([
            f"- {name}({cfg['title']}): 在{cfg['location']}{cfg['activity']},性格{cfg['personality']}"
            for name, cfg in self.npc_configs.items()
        ])
        return f"""请为Datawhale办公室的3个NPC生成当前的对话。

【场景】{context}
【NPC信息】{npc_desc_text}

【输出格式】严格按照JSON: {{"张三": "...", "李四": "...", "王五": "..."}}

【示例】{{"张三": "这个bug见鬼了,调试两小时了...", "李四": "这个功能优先级需要重新评估。", "王五": "咖啡拉花不错,灵感来了!"}}"""
```

## 架构决策对比

| 决策 | 选择 | 原因 |
|------|------|------|
| NPC 类型 | 独立 SimpleAgent（非共享） | 每个 NPC 有独立的记忆和人格 |
| 记忆系统 | Working + Episodic | 短期保持连贯，长期支持语义检索 |
| 背景对话 | 批量生成（一次 LLM 调用） | 成本 1/N，且同一次调用生成的内容更协调 |
| 交互对话 | 即时响应（单独调用） | 保证个性化和准确性 |
| 状态持久化 | SQLite + Qdrant | SQLite 存结构化状态，Qdrant 存向量检索 |

## 关键挑战

- **通信协议**：NPC 间通过共享环境和状态管理器间接通信（非直接对话）
- **任务分配**：批量生成器中由单一 prompt 驱动所有 NPC，无显式任务分配
- **成本控制**：混合模式（批量 + 即时）是关键策略——批量生成做"背景噪音"，即时响应用在关键交互
- **状态一致性**：`state_manager` 统一管理所有 NPC 状态，后端单点保证一致
