# 第03章 线性表

## 一句话总结

线性表是零个或多个元素的有限序列；选择顺序表或链表取决于随机访问、局部插删、容量和链接开销，而不是名称偏好。

## 学习目标

- 掌握前驱/后继关系、空表/表长语义及线性表 ADT。
- 解释顺序表地址计算、插删搬移，和单链表定位、指针重连。
- 区分头指针/头结点、静态、循环和双向链表所解决的问题。

## 核心概念与严格定义

- **线性表**是零个或多个数据元素的有限序列；非空表中除首元素外都有唯一直接前驱，除尾元素外都有唯一直接后继。
- **顺序存储**把元素置于连续单元，可按首地址、下标和元素大小计算地址；表长是现有元素数，数组长度是容量，二者不同。
- **单链表**节点由数据域和后继指针构成；头指针指向首节点，头结点是可选附加节点，用于统一空表和首节点操作。

## 关键结构、算法或操作流程

1. 顺序表取值先检查 `1 ≤ i ≤ length` 再定位；插入先判位置/容量、由尾向后搬移后写入；删除保存目标、将后继前移并减表长。
2. 单链表按位取值需从头顺链定位；插入先令新节点指向原后继再改前驱链接；删除先保存目标链接再跨过目标节点。
3. 静态链表用数组游标模拟链接；循环链表让尾链接回头部；双向链表增加前驱链接，以空间换双向移动和已知节点处的删除便利。

## 复杂度、前提条件和适用边界

- 顺序表按下标访问为 `O(1)`，中间插删最坏搬移 `O(n)`；适合随机访问多、规模较稳定的情形。
- 单链表定位为 `O(n)`，已定位到前驱后插删可为 `O(1)`；若只给位置序号，定位成本不能省略。
- 链表有指针字段、分配和局部性代价；C90 `malloc`/指针是实现细节，稳定要点是边界检查和链接不变式。

## 典型例子与易错点

- 例：顺序表插入应由尾倒序搬移，避免覆盖未搬走元素；链表插入应先保存原后继。
- 易错：混淆数组容量与表长，访问未初始化位置或错误判满。
- 易错：把“链表插删 `O(1)`”说成无条件结论。
