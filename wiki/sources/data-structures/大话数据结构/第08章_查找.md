# 第08章 查找

## 一句话总结

查找是在查找表中定位关键字相等的记录；性能由有序性、更新频率、分布、树高、索引和散列冲突共同决定。

## 学习目标

- 区分静态/动态查找表、关键字/记录及成功/失败查找成本。
- 比较顺序、二分、插值、Fibonacci 查找和稠密/分块/倒排索引的前提。
- 掌握 BST、AVL、2-3/2-3-4/B/B+ 树和散列表维护的结构不变式。

## 核心概念与严格定义

- **查找**是按给定值在查找表中确定关键字等于给定值的数据元素（记录）的过程；静态表主要查找，动态表还需插删。
- **BST** 对每节点保持左子树关键字小于根、右子树大于根（重复键策略须明确）。
- **AVL 树**任意节点左右子树高度差绝对值不超过 1，靠旋转恢复平衡。
- **散列表**以散列函数把关键字映射到地址；不同键同址是冲突，装填因子影响查找性能。

## 关键结构、算法或操作流程

1. 顺序查找逐项比较，可用哨兵减少边界判断；二分在有序区间比较中点；插值按分布估计位置，Fibonacci 按 Fibonacci 分割位置。
2. 稠密索引为每记录配索引项，分块索引块间有序而块内可无序，倒排索引由关键字指向记录/文档集合。
3. BST 沿比较方向下行；插入在空子树落点；删除区分叶、单子树、双子树。AVL 按 LL/LR/RR/RL 失衡情形旋转。
4. B 类树用多关键字、多子节点控制高度；B+ 树把记录/指针组织在叶层并连接叶。散列表先构造函数，再选开放定址、再散列、链地址或公共溢出区处理冲突。

## 复杂度、前提条件和适用边界

- 顺序查找最坏 `O(n)`；二分、插值、Fibonacci 查找要求有序表和有效中点定位。插值效果依赖关键字近似均匀分布。
- BST 性能依树高，退化为链则最坏 `O(n)`；AVL/B 类树以平衡或多路分支控制高度，但更新更复杂。
- 散列表平均性能依装填因子、均匀性和冲突策略，最坏可退化；C90 的表长、空槽、取模函数是策略细节，稳定概念是映射和冲突解决。

## 典型例子与易错点

- 例：二分查找不仅要有序，还要低成本定位中点；普通链表反复定位中点会失去优势。
- 易错：把倒排索引理解为“原表倒过来”；它是词项/关键字到记录集合的映射。
- 易错：认为哈希表恒为 `O(1)` 或天然有序；冲突/装填决定平均成本，且并不提供排序语义。
